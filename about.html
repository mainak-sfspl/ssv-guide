What Problem SSV Solves

Most security incidents don’t start with “hackers” — they start with credentials scattered everywhere: API keys in code, passwords in chat, and shared spreadsheets nobody can audit.

  • Multi‑server credential sprawl: secrets spread across servers, scripts, and tools.
  • Personal multi‑password chaos: users juggle multiple passwords with no central oversight.
  • High risk outcomes: accidental exposure, data leaks, complex maintenance, and zero accountability.

The Objective

Sampurna Secure Vault (SSV) establishes a single, self‑hosted credential and secret vault to manage all passwords, API keys, certificates, and automation tokens — for both users and applications.

The Solution: Keycloak + HashiCorp Vault

SSV combines identity & access management (Keycloak) with secrets management (HashiCorp Vault) — keeping identity separate from secrets and enforcing segregation of duties.

  • Keycloak (IAM): user lifecycle, admin‑only creation, forced password changes, and MFA.
  • Vault (Secrets): secure storage, fine‑grained RBAC/policies, versioning + rotation, and audit logs.

How It Works (Core Workflows)

  1. New user onboarding: admin creates the user, shares temporary access, user logs in and enables MFA.
  2. Password reset (without email): helpdesk/admin resets password securely and forces change at next login.
  3. Secure secret retrieval for applications: apps fetch short‑lived tokens at runtime (no secrets hardcoded).

Business Impact

  • Zero secrets in code: removes accidental Git leaks and hardcoding.
  • MFA + Audit by default: every view/edit/delete becomes traceable.
  • Low operational overhead: initial setup typically needs only a small engineering effort, then lightweight maintenance.

✅ Bottom line: SSV reduces credential risk, improves compliance, and makes day‑to‑day operations predictable — without adding heavy process.
